How can we make science more accessible to a broader public, including non-expert readers from neighboring disciplines, students and the public with varying levels of expertise? What is the role of graphic design, visual communication, and other 'non-traditional' modes of scientific communication in our work?

I fully agree with the readings this week arguing for non-traditional modes of scientific communication! Well-crafted visuals and interactive demos can be so much more effective than traditional papers. Chris Olah (co-founder of Distill) had a huge effect on me personally. I remember reading his introduction to information theory in college, and to this day I remember that blog post better than all of the classes I took that covered information theory. I've since read all of Chris's posts, which always use graphics and interactive plots to great effect. Similarly, I'm a huge fan of Distill, and learned so much from the beautifully-crafted publications there. 3Blue1Brown is also a channel I've been a fan of for a long time --- I've watched all of their videos and some of them are permanently ingrained in my memory even years later (such as the beautiful and mind-blowing inscribed rectangle post).

I've long been inspired by Distill-style posts, and a few years ago even began drafting my own submission to Distill on variational autoencoders. Unfortunately I dropped that project because it was taking so much time, but I do hope to make interactive webpages for my future publications to explain concepts more clearly and to a broader audience than is possible with a traditional paper.

I wish more scientists would use animations and interactive plots to show their work. However, there are two major obstacles:

  • There is little incentive. Scientists are evaluated primarily based on publications in journals, particularly high-impact journals. And high-impact journals are still stuck in the traditional mode.
  • Making nice graphics and interactive webpages is time-consuming, often frustrating, and requires technical skills. For example, making the interactive plot on this page (see below) took more than two hours and involved programming in multiple langauges (in this case python, javascript, and html).
I think these obstacles can and should be overcome. For example:
  • Major journals should accept papers with animations and interactive plots. They could create a separate category for these (like review papers, articles, opinion papers, ...). I would absolutely submit an interactive paper if it were an option at the leading journals in my field!
  • MIT or the BCS department could offer resources to help us make interactive posts about our research. For example, imagine a small team of web and/or graphics designers that each week take one paper from the MIT community and make a blog post about it. That would be fantastic, and might benefit the broader public if the posts are interactive and accessible without assuming much background knowledge.

Commentary

The following section of this webpage is my draft of an interactive post. First, I'll say a few words about the tools I used. I used GitHub Pages to host this page. I really like GitHub Pages, because: (i) it is free, (ii) the code implementing the webpage is in a GitHub repository, which lends nice version control features. You can use the same GitHub repo for code, data, and webpage. This makes it a one-stop shop for a research project: When you open-source the code and data for a paper, you can have a webpage with interactive plots in the same repo and GitHub will host it.

To make the interactive plot below, I used javascript. Specifically, I used Plotly.js, which has a bunch of open-sourced tools for embedding plots in html webpages. For the data, I pushed a dataset (from a research project) to the same GitHub repo as the code for this webpage, which is loaded by the javascript code to make the plots below.

Overall, making this website was not difficult. I'm not an expert at html or javascript so I had to spend a little time debugging, but with practice this would probably become very easy, like Latex or Markdown. I'm now seriously considering making an interactive website for the next paper I write.

Factorized Representations for Generalization

There's a lot of text I'd write here motivating and describing the project, since my intended audience would be broad (anyone with high school level math). But for this assignment I'm just going to focus on the visualizations and interactive plot, so won't describe the science at all. I'll just mention that we define a "factorized" embedding \(f\) to be one that decomposes as a linear sum \(f(x, y) = f_x(x) + f_y(y)\). We then show that factorized embeddings facilitate generalization of neural network models to novel stimuli. Moreover, factorization seems to be exactly the criteria that best predicts generalization, as opposed to linear decodability of factors, orthogonality, disentangling, or other measures of structure in the embedding.
